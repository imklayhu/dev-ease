# DevEase · 竞品与关键词隙（定性）

> 说明：未接入 DataForSEO 等 API，以下为 **桌面研究 + 行业常识** 定性结论，用于选题与差异化；具体搜索量与难度请以 Google Search Console / 关键词工具校准。

---

## 1. 参照竞品（非穷尽）

| 竞品/形态 | 典型特征 | 可借鉴 | DevEase 差异化 |
|-----------|----------|--------|----------------|
| **JSON Formatter / jsonformatter.org** 等 | 单工具流量大、英文为主、广告多 | 清晰的输入/输出区、复制按钮 | 中文 UI、多工具一体、无广告叙事 |
| **IT-Tools / 类聚合开源** | 工具数量多、GitHub 可信 | 工具分类、开源背书 | 更轻的部署、品牌化与视觉统一 |
| **SmallDev.tools 等** | 现代 UI、多工具 | 卡片索引、单页专注 | 本地优先、操作历史、关于页数据透明 |
| **各大云厂商文档内嵌工具** | 权威、长尾词强 | 技术准确、术语覆盖 | 中立第三方、不绑定云账号 |
| **VS Code 插件 / 本地 CLI** | 非网页 SEO | — | 网页「免安装」场景词 |

## 2. 内容策略启示（来自 saas 模板）

- **对比型查询**：`X 在线工具 推荐`、`JSON 格式化 网站` — 可通过一篇 **聚合对比/清单** + 内链到本站工具承接。
- **场景型查询**：`JWT 调试 不写服务器`、`Base64 图片 前端` — 适合 **短指南** + 指向对应 `/tools/.../`。

---

## 3. 关键词隙（建议优先队列）

**高意图 / 与现有工具强相关**

- JSON 格式化、JSON 压缩、JSON 校验  
- Base64 编码解码、URL 编码  
- Unix 时间戳 转换、UUID 生成  
- SHA256 在线、正则 测试 在线  
- JWT 解码（强调浏览器端、不验签）  
- 颜色 HEX RGB 转换  
- 二维码 生成 文字  

**中长尾 / 需内容支撑**

- 「纯前端」「无需上传」「隐私」+ 工具名（与品牌定位一致）  
- 「GitHub Pages 部署 工具站」（品牌/开发者向）  

**英文（可选第二阶段）**

- 若增加 `en` 路由或独立路径，可抢 **global long-tail**；需 `hreflang` 与独立 meta。

## 4. E-E-A-T 对标观察

竞品常见强信号：**GitHub 仓库星标、更新日期、作者/团队页、安全说明**。  
DevEase 建议：在关于/settings 区域强化 **仓库链接、开源协议、数据不离开本机** 的可验证陈述，并与 JSON-LD `Organization` / `WebSite` 对齐（见路线图）。
