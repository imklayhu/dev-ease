# 批量小工具（v2）原始需求证据

## 摘要

在「开发者工具集合站」定位下，补充高频、可纯前端实现、无需后端的通用能力，覆盖数据格式、编码与时间场景，降低在多个站点间切换的成本。

## 证据表

| 需求点 | 原始引述/观察 | 来源 | 置信度 | 待验证项 |
|---|---|---|---|---|
| 需要更多小工具 | 用户要求「调研更多的小工具，按既定流程实现」 | 用户直接陈述 | 高 | 首批工具优先级是否调整 |
| 形态约束不变 | 项目宪章：纯前端、GitHub Pages、IndexedDB | `AGENTS.md` | 高 | 无 |
| 典型开发者诉求 | JSON 校验/格式化、Base64、时间戳、UUID 为常见在线工具 | 行业常见用法/推断 | 中 | 是否需要「隐私模式/不落盘」提示 |

## 竞品/替代方案简表

| 类型 | 代表 | 启发 |
|---|---|---|
| JSON 工具站 | jsonformatter 等 | 需要清晰错误信息与格式化/压缩 |
| 编码工具 | CyberChef / 小工具站 | 编解码需 UTF-8 安全 |
| 时间戳站 | epoch converter | 秒/毫秒与本地时区展示 |

## 本轮非目标

- 后端存储、账号体系、跨设备同步（除浏览器本地 IDB 外）。
- 大文件/流式处理与复杂 diff（保持轻量）。

## 开放问题

1. 是否需要「输入不落服务器」的更显著提示（纯前端已满足，但可增强文案）。
2. 下一批：哈希（SHA-256）、正则测试、颜色转换等。
