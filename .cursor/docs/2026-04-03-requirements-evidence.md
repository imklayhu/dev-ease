# dev-ease 原始需求证据（MVP 第 1 轮）

## 摘要

当前目标是启动一个可持续扩展的纯前端工具集合站，先完成可上线的基础壳与首个示例工具能力。核心诉求是：低部署成本（GitHub Pages）、可持续扩展（Next.js App Router）、可本地持久化用户偏好与行为（IndexedDB）。

## 证据表

| 需求点 | 原始引述/观察 | 来源 | 置信度 | 待验证项 |
|---|---|---|---|---|
| 需要纯前端形态 | “项目是一个纯前端页面的开发或者其他便利工具的集合站” | 用户直接陈述 | 高 | 工具范围是否仅开发类 |
| 使用 GitHub Pages 发布 | “使用github pages发布” | 用户直接陈述 | 高 | 是否使用自定义域名 |
| 前端框架为 Next.js | “前端使用nextjs” | 用户直接陈述 | 高 | 是否固定 TypeScript |
| 需本地持久化 | “希望可以使用indexdb存储一些用户行为和设置” | 用户直接陈述 | 高 | 行为数据边界（匿名颗粒度） |
| 需要流程化协作 | 用户连续要求创建 market/pm/dev-test subagent 与 harness 约束 | 对话行为观察 | 高 | 是否引入版本化模板 |

## 竞品/替代方案简表（轻量）

| 类别 | 代表 | 核心承诺 | 对本项目启发 |
|---|---|---|---|
| 在线开发者工具聚合 | 工具导航类站点 | 快速找到并使用工具 | 首屏要有清晰入口与分类 |
| 单功能工具站 | JSON/时间戳/编码类小站 | 专注单问题高效率 | 每个工具页要独立、可收藏 |
| 本地优先 Web App | 设置可记忆应用 | 重启后保持状态 | IDB 存储层需稳定与可迁移 |

## 当前轮次非目标

- 不引入后端数据库与鉴权系统。
- 不做复杂账号体系与跨端同步。
- 不在第一轮引入重型设计系统。

## 开放问题与建议验证

1. 工具优先级：首批 3-5 个工具具体名单。
2. 行为数据范围：仅本地统计还是后续支持导出。
3. 建议验证动作：
   - 与潜在用户确认“最常用工具 Top 5”；
   - 试运行首页 + 一个工具页，收集可发现性反馈。
