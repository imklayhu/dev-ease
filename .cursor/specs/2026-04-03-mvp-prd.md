# dev-ease MVP PRD（v0.1）

## 1. 背景与目标

- 背景：需要一个可长期维护的纯前端工具集合站，部署在 GitHub Pages。
- 目标用户：开发者与高频使用线上小工具的技术从业者。
- 本期目标：完成“可发布基础站点 + 首个示例工具 + IDB 基础持久化”。
- 非目标：账号体系、云端同步、后端服务。

## 2. 需求列表

| 编号 | 描述 | 优先级 | 价值 | 可行性 | 风险/假设 |
|---|---|---|---|---|---|
| R1 | 创建 Next.js 工程并支持 GitHub Pages 静态发布 | Must | 能上线 | 高 | 需配置 `output: 'export'` 与 `basePath` |
| R2 | 首页展示项目定位与示例工具入口 | Must | 可访问 | 高 | 初期 UI 简洁即可 |
| R3 | 提供“设置”能力并持久化到 IndexedDB | Must | 用户体验连续性 | 高 | 需 IDB 不可用降级 |
| R4 | 记录匿名本地行为（最近访问时间、访问次数） | Should | 后续优化依据 | 高 | 不采集 PII |
| R5 | 预留工具扩展结构（目录与组件） | Must | 可扩展性 | 高 | 先实现 1 个示例工具 |

## 3. 用户场景

- Given 用户首次访问首页，When 点击示例工具入口，Then 成功进入工具页并完成一次操作。
- Given 用户修改主题偏好，When 刷新页面，Then 设置仍被保留。
- Given 用户多次访问工具页，When 查看页面信息，Then 可读取本地访问计数。

## 4. 功能说明

1. 首页
   - 展示站点名称、说明、示例工具卡片（含进入链接）。
2. 示例工具页（MVP: 文本计数器）
   - 输入文本，实时显示字符数、单词数、行数。
3. 设置模块
   - 支持主题模式（light/dark/system）写入 IDB。
4. 行为记录
   - 记录工具访问次数、最近访问时间（本地 IDB）。
5. 异常处理
   - IDB 不可用时提示并回退到内存态（本次会话可用）。

## 5. 数据与配置

- DB 名称：`devEaseDB`
- 版本：`1`
- Stores：
  - `settings`（keyPath: `key`）
  - `activity`（keyPath: `toolId`）

## 6. 非功能要求

- 可通过 `next build` 生成静态产物。
- 路由兼容 GitHub Pages 子路径。
- 代码结构清晰，便于新增工具。

## 7. 验收标准（AC）

- AC1：执行构建成功并生成静态导出目录。
- AC2：首页可进入示例工具页。
- AC3：示例工具页可完成文本统计。
- AC4：主题设置刷新后仍生效（在同浏览器）。
- AC5：工具访问行为可写入并读取（至少次数递增可见）。
- AC6：IDB 不可用时页面不崩溃，存在降级提示。

## 8. 里程碑

- M1：工程初始化 + Pages 配置。
- M2：首页与示例工具页。
- M3：IDB 封装与设置/行为接入。
- M4：验收与构建验证。
