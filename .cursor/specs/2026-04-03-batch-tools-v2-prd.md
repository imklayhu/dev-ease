# 批量小工具（v2）PRD

## 背景与目标

- 扩充 dev-ease 的工具矩阵，覆盖开发/排障常见场景。
- 目标用户：开发者与技术从业者。
- 非目标：引入后端服务；处理超大文本的性能优化（后续迭代）。

## 需求列表

| 编号 | 描述 | 优先级 | 可行性 |
|---|---|---|---|
| T2-1 | JSON：格式化/压缩/校验（错误可读） | Must | 高 |
| T2-2 | Base64：UTF-8 安全编码/解码 | Must | 高 |
| T2-3 | 时间戳：秒/毫秒 ↔ 可读时间（本地时区） | Must | 高 |
| T2-4 | UUID：批量生成 v4（`crypto.randomUUID`） | Must | 高 |
| T2-5 | URL：encodeURIComponent / decodeURIComponent | Should | 高 |
| T2-6 | 统一壳层：访问记录、面包屑、返回工具区 | Must | 高 |

## 验收标准（AC）

- AC1：每个工具均有独立路由，并可从首页工具区进入。
- AC2：每个工具页记录本地访问（沿用 `activity` store）。
- AC3：JSON：能格式化与压缩；非法 JSON 显示错误信息（不崩溃）。
- AC4：Base64：中文等 UTF-8 文本编解码正确。
- AC5：时间戳：支持秒/毫秒输入歧义的基本规则（见实现说明）；展示 ISO 与本地时间。
- AC6：UUID：可一次生成多条并支持复制。
- AC7：`npm run build` 静态导出成功。

## 里程碑

- M1：共享壳层（hook + 访问面板 + header）。
- M2：实现 5 个工具页。
- M3：首页与导航更新 + 构建验证。
