# 关于我们 · 本地使用习惯与行为统计

## 背景与目标

用户在「关于我们」页需要**感知自己的使用习惯**，增强「本地优先、数据在我手里」的信任感；统计**仅来自本机 IndexedDB**，不上传。

## 数据范围（MVP）

| 数据源 | 字段 | 说明 |
|--------|------|------|
| `activity` 对象仓库 | `toolId`, `count`, `lastVisitedAt` | 已有：每次进入工具页 `recordToolVisit` 累加 |
| `settings` | `theme` | 展示**当前**主题偏好（非历史） |

不在 MVP 内：按日/按小时趋势、主题切换历史、跨设备同步（需新 schema 与写入点）。

## 功能需求

1. **概览 KPI（3 项）**  
   - 累计访问次数（所有工具 `count` 之和）  
   - 有记录的工具数量（`count > 0` 的条目数）  
   - 全站最近一次访问时间（所有 `lastVisitedAt` 的最大值）

2. **工具维度图表**  
   - 横向条形图：各工具访问次数，**降序**；条宽 = 相对最大值比例  
   - 显示工具**展示名**（与首页注册表对齐）；未知 `toolId` 回退为 id  

3. **分类维度图表**  
   - 按 `src/data/tools.ts` 的 `categoryId` 汇总访问次数  
   - 横向堆叠感：每类一条 bar，或分段条（本实现：每类独立 bar + 占比说明）

4. **空状态**  
   - 无任何活动记录时：说明文案 + 引导链到 `/#tools`

5. **隐私与说明**  
   - 固定短文案：数据仅存本机、清除站点数据会丢失、与账号无关  
   - 各工具页另有 **操作历史**（`toolHistory` 时间线），与本页访问统计互补；规格见 `2026-04-03-tool-page-history.md`。

## 交互与视觉（ui-ux-pro-max 方向）

- **信息层级**：概览 → 工具排行 → 分类分布 → 当前主题，单页纵向滚动。  
- **质感**：与站点一致的 glass、渐变描边、紫/琥珀强调；图表用 CSS 条形 + `tabular-nums`，避免引入大型 chart 依赖。  
- **动效**：条形 `transition-[width]`，`prefers-reduced-motion` 时缩短或关闭。  
- **无障碍**：`section` + `aria-labelledby`；条形区域 `role="img"` + `aria-label` 描述数值；保留文字数字供读屏。

## 技术实现要点

- 新增 `getAllToolActivities()` 只读遍历 `activity` store。  
- 客户端组件在关于页拉取活动 + 当前主题，用纯函数聚合。  
- 关于页 `max-w` 可略放宽以容纳图表（如 `max-w-4xl`）。

## 验收

- 有访问记录时 KPI 与条形图与 IDB 一致；清空 IDB 后显示空状态。  
- 无 IDB 时降级不崩溃（内存态或空）。  
- `npm run lint` / `npm run build` 通过。
